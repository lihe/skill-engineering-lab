# 项目OnePage

## 一句话

Skill Engineering Lab 是一个 AI Agent Skill 工程化评测实验室，用同一批样例对比无 Skill、Skill v1、Skill v2，自动生成触发、质量、效率和治理报告，展示 Skill 如何从提示词沉淀为可回归的团队能力资产。

## 我解决的问题

很多 Agent Skill 看起来能跑，但上线后常见三个问题：

- 触发不稳定：该触发时漏掉，不该触发时误触发。
- 质量不可证：只能展示单次好结果，无法证明相对基线有净增益。
- 无法治理：失败样例、版本假设和回归集没有沉淀，后续越改越难维护。

这个项目把这些问题转成一个可运行的工程闭环。

## 我做了什么

- 设计了 `without_skill -> with_skill_v1 -> with_skill_v2` 的对照评测流程。
- 构造了 12 条结构化样例，覆盖核心、边界、扩展和负向场景。
- 实现了确定性运行器、规则评分器、报告生成器和输出校验器。
- 将触发、产出、风格、效率、泛化拆成可诊断指标。
- 将失败样例沉淀为版本记录、根因归档和回归集。

## 结果

| 指标 | 无 Skill | Skill v1 | Skill v2 |
| --- | ---: | ---: | ---: |
| 通过率 | 16.7% | 75.0% | 100.0% |
| 触发召回率 | 不适用 | 80.0% | 100.0% |
| 过度触发率 | 不适用 | 50.0% | 0.0% |
| 平均 Token | 720 | 1142 | 935 |

v2 相比 v1，在质量提升 25.0% 的同时，平均 Token 下降 18.2%。

## 技术栈

- Python 3.10+
- JSON / JSONL 结构化评测资产
- Markdown 报告生成
- 规则评分器
- 确定性输出校验器
- GitHub 开源项目组织

## 简历写法

可以写成：

> 设计并实现 Agent Skill 工程化评测实验室，构建 12 条多类型评测样例与确定性评分 Harness，对比无 Skill、Skill v1、Skill v2 三种状态，自动生成触发召回、过度触发、质量提升、Token 成本和失败归因报告。通过收紧 description、硬化产出契约和引入脚本化校验，将示例 Skill 通过率从 75.0% 提升到 100.0%，过度触发率从 50.0% 降至 0.0%，并沉淀版本记录、回归集和根因归档，形成可复用的 Skill 治理闭环。

## 面试讲法

如果面试官问“这个项目难点是什么”，可以答：

> 难点不在于生成视频文案，而在于如何证明 Skill 真的有价值。我把问题拆成触发、产出、风格、效率、泛化和治理，避免只看单次输出。然后用 without_skill / with_skill_v1 / with_skill_v2 做对照实验，让每个失败都能回到 description、SKILL.md、references 或 scripts 的具体修改点。

如果面试官问“这个项目还能怎么扩展”，可以答：

> 当前版本用确定性模拟保证可复现，下一步可以接入真实 Agent 调用，保留相同评分接口；再加 HTML dashboard 和 benchmark history，用于跟踪多版本、多模型和多 Skill 的长期表现。
