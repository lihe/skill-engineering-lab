# Cover Style

Use this reference when generating thumbnail cover text for short-form AI product videos.

## Core Principle

Cover text is a title-like first-glance result phrase.

The title explains the hook. The cover compresses that hook into a harder two-line or three-line phrase.

Do not write a flat feature label. Start from a Jack-Cui-style title hook, then shrink it.

Preferred logic:

1. Show the visible result.
2. Add a small tension point: old action, surprise, plug-in metaphor, role reversal, or creator-tested feeling.
3. Compress it into 2-3 short lines.
4. Make the viewer understand the value in one glance.
5. Keep product names secondary unless the brief explicitly requires main-text exposure.

## Title-Cover Relationship

Use different jobs:

- Title: full hook, context, tension, and promise.
- Cover text: a title-like compression of the sharpest result, contrast, or capability.

Good pairing:

- Title: 别再手写日报了，AI Agent 已经能自己整理工作记录
- Cover: 别再手写日报了 / 让智能体自己接管

Avoid:

- Pasting the full title onto the cover.
- Writing a complete sentence with too many clauses.
- Making the product name the only hook.
- Putting hashtags, CTA, download links, or platform notes on the cover.

## Line Rules

Default cover text:

- 2-3 lines.
- Include both 2-line and 3-line candidates when ideating.
- 2-line candidates should use fuller title-like lines: 7-10 Chinese characters per line.
- 3-line candidates should use tighter lines: 5-7 Chinese characters per line.
- Vary line lengths within each candidate when possible, such as 7/9 for two lines or 5/6/7 for three lines.
- Avoid same-length stacks such as 8/8 or 6/6/6 when a natural staggered version exists.
- No 2-line line should exceed 10 Chinese characters; no 3-line line should exceed 7 Chinese characters.
- 2-line total length is usually 14-20 Chinese characters; 3-line total length is usually 15-21 Chinese characters.
- One idea per line.
- Use `/` in draft output to indicate line breaks.
- Prefer nouns, verbs, numbers, and short emotional judgments.
- It can feel close to the title's style, but should read more like a thumbnail headline than a sentence.

Good cover words:

- AI, Agent, Skill, 工作流, 自动化
- 一键, 到点, 自动, 本地, 实测, 保姆级
- 外挂, 隐藏用法, 真能干活, 不用手写, 直接跑
- 省步骤, 跑流程, 变日报, 变视频, 变应用
- 别再, 破防, 离谱, 接管, 这才是, 终于, 不是陪聊

Use these Jack-Cui-style hook moves before writing cover text:

- **旧动作拦截**: 别再 [旧动作] 了.
- **外挂比喻**: [场景/人群] 的 [任务] 外挂.
- **角色反转**: 不是 [旧AI角色], 是 [新执行角色].
- **结果震感**: [普通材料] 直接变 [高价值结果].
- **真能干活**: [AI/Agent] 终于能 [具体任务].
- **破防/离谱**: use only when the demo has a visible surprise; keep it less absolute than "最强" or "天花板".

Flat-to-hook rewrites:

- 自动写日报 -> 别再手写日报了 / 让智能体自己接管
- 到点生成 -> 下班前一键运行 / 日报直接写好了
- 工作记录很多 -> 一堆工作记录里 / 直接变成日报了
- 帮你干活 -> 别再只让它聊天 / 现在真能干活了
- 自动工作流 -> 日报外挂真来了 / 重复活都接管了

Commercial safety:

- Replace unsupported "99%的人不知道" with "很多人可能还没注意".
- Replace unsupported "最强/天花板/吊打/碾压" with "很能打", "有点意外", "这次真能用", or a specific demo-backed result.
- Avoid "唯一/绝对/最安全/全网首发" unless the brief proves and permits it.

## Cover Formulas

### 1. Old-Action Interception

Use when the video can stop a familiar painful action.

Templates:

- 别再 [旧动作] / [新结果]
- [痛点动作] / 还要手动吗
- [重复动作] / 这次能省了

Examples:

- 别再手写日报了 / 让智能体自己接管
- 每天写日报崩溃 / 现在不用手动了
- 下班还补日报吗 / 这次能直接省了

### 2. Plug-in Feeling

Use when the product makes an old workflow feel unusually easier.

Templates:

- [人群/场景] 外挂
- [工具] / 像开了外挂
- [重复工作] / 终于能自动跑

Examples:

- 日报外挂真来了 / 下班不用补日报
- 打工人真破防了 / 日报被智能体接管
- 电脑里的AI外挂 / 终于开始干活了

### 3. Role Reversal

Use when the product changes AI from chat to execution.

Templates:

- 不是 [旧角色] / 是 [新角色]
- [AI] 不只 [旧动作] / 还能 [新动作]
- [旧认知] / 这次变了

Examples:

- 不是陪你聊天了 / 而是直接干活了
- 别再只问问题了 / 让它直接动电脑
- 聊天智能体够吗 / 现在能操作电脑

### 4. Result Shock

Use when the demo has a clear before/after transformation.

Templates:

- [旧材料] 很多 / [新结果] 直接出
- [普通输入] / 变成 [高价值结果]
- [动作] 一下去 / [结果] 就出来

Examples:

- 一堆工作记录里 / 直接变成日报了
- 一句话下去以后 / 日报自己跑出来
- 浏览记录全翻出 / 日报当场生成了

### 5. Low-Barrier Tutorial

Use when the video teaches a setup, Skill, workflow, or tool chain.

Templates:

- 手搓一个 / [结果] Skill
- 从0做一个 / [工作流]
- 保姆级 / [任务] 工作流

Examples:

- 手搓日报技能流 / 下班自动运行了
- 从零做个日报流 / 以后每天自动跑
- 手搓一个技能后 / 日报到点自己出

### 6. Capability Contrast

Use when the product changes the role of AI.

Templates:

- [旧角色] / [新角色]
- [输入] / 自动变 [输出]
- 不是 [旧定位] / 是 [新能力]

Examples:

- 不是聊天智能体 / 而是执行智能体
- 工作记录堆满了 / 自动变成日报了
- 别再陪你聊天了 / 直接帮你干活

### 7. Time Or Number Proof

Use only when the brief or demo supports the number.

Templates:

- [数字] 秒 / 生成 [结果]
- [时间] / 跑完 [任务]
- [数量] 步 / 变 [结果]

Examples:

- 三分钟做个技能 / 下班前自动跑
- 一条指令下去后 / 日报自动生成了
- 五步跑完流程后 / 日报自己出来了

## Product-Type Guidance

### AI Agent / Workflow

Prioritize:

- 真能干活.
- 操作电脑.
- 自动跑流程.
- 可复用 Skill.

Cover directions:

- 不是陪你聊天了 / 而是直接干活了
- 自动工作流跑起 / 重复活都接管了
- 手搓日报技能流 / 下班自动运行了

### AI Coding / App Builder

Prioritize:

- From idea to runnable output.
- Low-code or no-code.
- Real build result.

Cover directions:

- 一句话做应用后 / 成品直接跑起来
- 没写一行代码 / 成品也能跑通
- 让AI来写代码 / 这次真能交付

### AI Video / Image / Creative Model

Prioritize:

- Visual transformation.
- Before/after.
- Speed and quality.

Cover directions:

- 图片直接变视频 / 这次真能用了
- 草图直接变大片 / 质感终于出来
- 这次AI生成物 / 终于能进工作流

### Cloud / Infrastructure

Prioritize:

- Cost, speed, deployment, stability.
- Translate technical terms into user-facing outcomes.

Cover directions:

- 不用自己搭环境 / 直接就能跑起来
- 算力成本太高了 / 这次真省下来
- 大模型部署这事 / 终于变简单了

### Learning / Career / Community

Prioritize:

- Low barrier.
- Roadmap.
- Opportunity.

Cover directions:

- 保姆级学习路线 / 入门终于不绕路
- 大厂AI技能点 / 现在就能学起来
- 现在开始学AI / 还来得及吗

## Output Pattern

When asked for cover text, output:

```markdown
## 封面文案

### 两行版
1. ...
2. ...

### 三行版
1. ...
2. ...

### 推荐

### 与标题的配合
```

For a full package, include cover text after the recommended title and before the outline.
