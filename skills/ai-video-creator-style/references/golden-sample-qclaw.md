# Golden Sample: QClaw Skill Workflow Script

Use this reference when judging whether a generated script matches the established creator口播 style. This is the current standard sample, not a product template.

## When To Read

Read this file when:

- Writing or revising a full 3:00-3:30 script.
- The user asks whether a script "像不像我" or "像不像标准样".
- Calibrating a new AI tool / Agent / workflow brief against the preferred口播 rhythm.
- Doing the final style check after title, outline, and script are complete.

Do not copy QClaw-specific claims into unrelated briefs.

## What This Sample Proves

This sample works because it:

- Opens by pointing at the visible result: "你现在看到的这份..."
- Talks to the viewer instead of explaining from a distance.
- Shows result first, then pain, then product intro.
- Introduces the product only once in part 1.
- Uses a result-to-tool handoff: "帮我把这套流程做出来的，是 QClaw."
- Avoids client-facing phrasing in口播; use "替你跑办公流程" instead of "接入办公场景".
- Extends the single demo into a reusable workflow, such as daily report to weekly/monthly report.
- Ends part 1 by promising a process walkthrough.
- Uses step markers in part 2: "首先", "接下来", "最后再到".
- Uses inclusive操作口吻: "我们还可以继续..."
- Closes the workflow before insight: "这样，整个skill流程就制作完毕了。"
- Elevates from surface result to workflow value.

## Style Calibration Checks

Before calling a new script final, compare it with this sample:

- First line: does it feel like the creator is pointing at the screen with the viewer?
- Part 1: does it avoid duplicate product reveal?
- Part 1: does the product reveal feel earned by the visible result rather than inserted as an ad line?
- Part 1: does it use creator-native office language rather than client-facing product wording?
- Part 1: if the demo is a single task, does it briefly point toward a longer-term workflow?
- Part 1 ending: does it bridge into the exact thing part 2 will demonstrate?
- Part 2: does it feel like a process walkthrough, not feature narration?
- Part 2: does each product capability appear inside a concrete step?
- Part 2: is there one iteration or constraint change?
- Part 2 ending: does it close the workflow before the value judgment?
- Part 3: does it move from feature to reusable workflow or creation method?

## Standard Script

### 第一部分｜产品引出

你现在看到的这份详细日报，是 AI 到点自动生成的。

今日完成、推进中、风险同步、明日重点，它都已经分好了。

重点是，我没有临下班才开始写。

也没有一条条去查文档修改记录和工作文件。

我只是提前做了一个自动日报 Skill。

到了固定时间，它就能自己运行，自己整理，自己输出。

这不是那种只能陪你聊天的 AI。

而是真的能替你跑办公流程。

有了这个 Skill，

AI 就能把写日报这件枯燥但重要的事，真正接过去。

不只是日报。

它还能继续沉淀成周报、月报。

帮我把这套流程做出来的，是 QClaw。

简单说，它是腾讯官方出品的本地 AI 助手。

和普通聊天 AI 不一样，它能在电脑上执行任务、处理文件、创建 Skill，并长期复用。

接下来，我就带你跑一遍日报 Skill 制作流程。

### 第二部分｜亮点 + 实测展示

首先打开 QClaw，直接输入一句话：

帮我创建一个自动日报 Skill。

要求很简单。

每天整理浏览器访问记录、文档修改记录和工作文件。

最后输出今日完成、进行中、问题风险和明日计划。

你可以看到，它不是简单回我一段建议。

它会先拆任务。

日报应该有哪些栏目。

哪些记录可以作为工作痕迹。

最后再到这套流程怎么保存成一个可以复用的 Skill。

这一步很关键。

因为一次性 prompt，今天能用，明天还要重新说。

但 Skill 不一样。

它更像是把你的工作方法保存下来。

接下来我直接运行一次。

QClaw 会根据授权的本地记录，把今天查过的资料、改过的文档、推进过的任务整理出来。

零散信息会被重新归类。

最后变成一份结构化日报。

我们还可以继续加一个限制：

口吻更职场一点，增加“风险同步”和“明日重点”。

它会在原来的 Skill 上继续改。

也就是说，skill不是一次生成完就结束。

而是可以一轮一轮迭代成你自己的日报工作流。

最后再设置定时任务。

每天 18:30 自动运行这个 Skill。

这样，整个skill流程就制作完毕了。

我觉得，这里最有意思的不是自动生成日报。

而是从创建 Skill，到运行 Skill，再到定时执行，整个过程都被串起来了。

这就和普通聊天 AI 拉开了差距。

以前写日报，是我下班前重新拼一天。

现在变成 QClaw 按固定流程帮我整理一天。

### 第三部分｜价值总结 + 行动号召

到这一步，其实已经很清楚了。

QClaw 最有意思的地方，不是它回答得多像人。

而是它能在本地电脑里执行任务、处理文件、创建 Skill、再把 Skill 放进定时工作流。

今天是日报。

明天也可以是周报、会议纪要、竞品分析。

如果你愿意继续加规则，它还可以记住你的格式、语气和固定栏目。

这才是 Skill 真正值钱的地方。

真正关键的不是让 AI 多聊几句。

而是把你每天重复做的事，沉淀成一个长期可复用的工作流。

也许办公自动化的下一步，不是问 AI 怎么做。

而是让 AI 到点自己做。

- This sample is a high-density version. For heavier visual demos, shorter scripts around 850-950 spoken characters are still acceptable.
